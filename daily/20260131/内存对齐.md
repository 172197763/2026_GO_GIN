发现了一个有趣的现象
```
i32_1 := int32(1)
i64_1 := int64(2)
fmt.Println(&i32_1, &i64_1)
i32_2 := int32(1)
i32_3 := int32(1)
i64_2 := int64(2)
fmt.Println(&i32_2, &i32_3, &i64_2)

//结果输出（各自运行环境会有差异）

0xc00008d1a0 0xc00008d1a8
0xc00008d1c0 0xc00008d1c4 0xc00008d1c8
```
首先这里打印的是变量的地址，可以看到<br>
i32_1第一位=0<br>
i64_1第一位=8<br>
i32_2第一位=0<br>
i32_3第一位=4<br>
i64_2第一位=8<br>
是什么造成了这种现象呢？<br>
这里就要说到内存对齐了<br>
什么是内存对齐呢？你可以设想一下。<br>
如果你有一个书桌<br>
第一层存放的是童话故事类型的书<br>
第二次存放的是历史文化的书<br>
这样找起来根据想要找的书的类型直接去到对应的层数逐本查找就可以快速找出来了<br>
摆放书籍的时候，就算某一层的书本满了，你会新起一层去存放，不会乱塞到其他类型的层数里面<br>
这样做都是为了可以快速地查找到你想要的书籍。<br>
